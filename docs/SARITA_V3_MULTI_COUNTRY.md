# SARITA v3.0: Lógica Multi-País y Jerarquía Territorial

## 1. Jerarquía Estructural
Para soportar la operación global, SARITA implementa una jerarquía de cuatro niveles:

1. **Plataforma Global (Root):** Control maestro, analítica agregada y estándares de red.
2. **País (Tenant):** Instancia nacional con su propio marco legal, moneda base y panel gubernamental.
3. **Región / Estado:** Subdivisión administrativa para gestión descentralizada.
4. **Ciudad / Municipio:** Unidad operativa base donde ocurre la interacción directa con prestadores y atractivos.

## 2. Configuración por País (Tenant Settings)
Cada país configurado en SARITA v3.0 define:
- **Idioma(s) oficial(es):** Para interfaces y comunicaciones.
- **Moneda Local:** Para el Ledger contable y transacciones POS.
- **Marco Normativo:** Reglas de impuestos (IVA, tasas turísticas), requisitos de verificación de prestadores (RNT o equivalente) y protocolos de seguridad.
- **Pasarelas de Pago Locales:** Integración con adquirentes y redes bancarias nacionales.

## 3. Registro Nacional de Prestadores (RNP)
SARITA v3.0 unifica el registro de prestadores a nivel nacional dentro de cada tenant, permitiendo:
- **Validación Institucional:** El gobierno nacional aprueba las categorías y estándares.
- **Visibilidad Global:** Un prestador registrado en el Tenant de Colombia es visible para turistas de cualquier otro Tenant.

## 4. Gobernanza Multinacional
- **Dashboards Nacionales:** Los Ministerios de Turismo acceden a su propia inteligencia territorial.
- **Cooperación de Datos:** Intercambio de métricas de flujo turístico entre países para análisis de tendencias migratorias y económicas.
