# PLAN DE CONVIVENCIA MULTI-REGULATORIA
**Gestión de Marcos Legales Divergentes**

## 1. CONTEXTO
Sarita opera en un entorno donde las leyes cambian por frontera. El sistema debe ser capaz de aplicar la Ley 1581 (Colombia) en un nodo y la GDPR (UE) en otro, de forma simultánea.

## 2. PARAMETRIZACIÓN NORMATIVA
Las leyes NO se programan en el código, se cargan como **Frameworks de Gobernanza**:
- **Framework Global:** Alineación mínima exigida (ej: Transparencia IA).
- **Framework Jurisdiccional:** Leyes específicas del país/región.
- **Framework Local:** Normas de convivencia turística municipal.

## 3. VERSIONAMIENTO LEGAL
El Kernel mantiene versiones de las leyes. Si una ley cambia, se aplica el nuevo Framework pero se puede revertir o auditar con el Framework anterior para procesos legales en curso.
