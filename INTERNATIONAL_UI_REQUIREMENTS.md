# REQUISITOS DE UI DIPLOMÁTICA INTERNACIONAL (INTERNATIONAL UI REQUIREMENTS)

**Versión:** 1.0 (Fase Z-INT)
**Módulo:** Dashboard Vía 1 (SuperAdmin)
**Propósito:** Proporcionar visibilidad total y control sobre las conexiones internacionales.

---

## 1. PANEL DE TRATADOS ACTIVOS (DIPLOMATIC HUB)
Interfaz centralizada para gestionar la federación:
- **Vista de Red:** Mapa global/regional mostrando los Nodos Nacionales conectados.
- **Lista de Tratados (TID):** Tabla con ID, País Socio, Fecha de Expiración y Nivel de Intercambio (N0-N2).
- **Control de Mandato:** Botón para cargar nuevos archivos TID firmados digitalmente.

## 2. MONITOR DE INTERCAMBIO REAL-TIME
Visualización de los datos que están cruzando la frontera:
- **Caudal de Datos:** Gráfico de volumen (MB) enviado/recibido.
- **Inspector de Eventos:** Vista tipo "Live Stream" mostrando la clasificación de cada paquete (ej: "Evento Nivel 1 - Agregado Estadístico").
- **Alertas de Integridad:** Indicador visual de la validez de los certificados mutuos (MTLS).

## 3. COMPONENTE DE REVOCACIÓN INMEDIATA (DIPLOMATIC KILL SWITCH)
Un botón de alta visibilidad (Institucional) que permite:
- **Corte Total:** Desactivar todas las conexiones federadas instantáneamente.
- **Corte Selectivo:** Suspender la cooperación con un país específico sin afectar a otros.
- **Feedback Visual:** Cambio de estado de "Conectado/Seguro" a "Aislado/Soberano".

## 4. ALERTAS TEMPRANAS REGIONALES (EARLY WARNING SYSTEM)
Sección compartida de inteligencia defensiva:
- **Threat Feed:** Notificaciones de ataques detectados por países socios bajo tratados de seguridad.
- **Nivel de Alerta Regional:** Indicador cromático (Verde/Amarillo/Rojo) basado en la consistencia de las señales internacionales.

## 5. REQUISITOS DE LENGUAJE Y UX
- **Terminología Diplomática:** Uso estricto del lenguaje institucional (Socio, Tratado, Mandato, Revocación).
- **No Ambigüedad:** Los estados de conexión deben ser binarios y claros (ej. No usar "Conectando..."; usar "Capa Federada: INACTIVA").
- **Auditabilidad:** Cada interacción del SuperAdmin con el panel internacional debe requerir confirmación y quedar sellada en la bitácora estatal.
